### What moves where

Core (new, header-only): declarativeimports/core/units/dropclassifier.h
namespace Latte::DropClassifier. Both consumers (containment +
plasmoid) share it, per spec.

1. MimeSnapshot value type: { QStringList formats; QString
   plasmoidServiceData (the "text/x-plasmoidservicename" payload as
   text); bool hasUrls; QList<QUrl> urls }. hasUrls and urls are BOTH
   carried: QMimeData::hasUrls() is format presence, urls() is parsed
   content, and the shipped JS reads both (guards on hasUrls, every()
   on urls).
   DEVIATION from the spec's interface sketch: taskData/sourceId
   dropped - no decision in either file reads the taskbuttonitem
   payload bytes or a source id; only format PRESENCE is tested.
   Carrying dead fields would imply decisions that do not exist.
2. classifyContainmentDrag(mime, isApplicationUrl) ->
   ContainmentDragFlags {isTask, isPlasmoid, isSeparator, isLatteTasks,
   onlyLaunchers} - DragDropArea onDragEnter's five decisions,
   including View::mimeContainsPlasmoid's newline-split name matching
   (separator/latte-tasks) vs the plasmoid's whole-payload comparison.
3. classifyTasksDrag(mime, isApplicationUrl) -> TasksDropFlags
   {movingTask, droppingOnlyLaunchers, droppingSeparator,
   droppingFiles} + tasksDragAccepts(flags) - MouseHandler
   onDragEnter's chain with its !movingTask / !onlyLaunchers
   precedence.
   isApplicationUrl is a synchronous callable seam (the EX-23
   edge-touch callback shape): the predicate consults live C++
   (extendedInterface/backend.isApplication) and stays outside the
   core.
4. Disposition ladders (the "decisions move where they cannot be
   silently disconnected" rule, b474adad):
   - containmentDragEnterAction(flags, immutable, viewShownFully,
     hasStealingApplet) -> Reject | ShowAddLaunchersMessage |
     InsertSpacer
   - containmentDragMoveAction(flags, hasStealingApplet) ->
     KeepCurrent | ShowAddLaunchersMessage | InsertSpacer
     (move never re-checks immutable/shownFully - faithful)
   - containmentDropAction(flags, viewShownFully, hasStealingApplet)
     -> Ignore | AddLaunchersToStealingApplet | ProcessMime
   - tasksDropAction(flags, hoveredItemIndex) -> Ignore |
     AddSeparator{position} | DropUrls (separator wins over
     launchers/files, faithful order; position = hovered index >= 0
     else 0)
5. insertIndexAt(aboveItemIndex, posInTarget, orientation, itemStep)
   -> int. Faithful to tools.js including the `above && above.itemIndex`
   TRUTHINESS quirk (index 0 falls through to stripe math) and the
   stripe-1 result that can go negative at/before the row origin
   (tasksModel.move rejects negatives; pinned as faithful, documented).
   DEVIATION from the spec's interface sketch: no RowEntries parameter.
   The shipped/Qt5 algorithm never consults the row - it returns the
   hovered delegate's index or uniform-step stripe math
   (metrics.totals.length). Fabricating a row snapshot the function
   ignores would be dishonest; no parallel type invented either. The
   spec's "insertIndexAt across separators/hidden" test slots are
   realized as hovered-separator / hovered-index cases.
6. decideTasksDragMove(snapshot) - MouseHandler onDragMove 154-190,
   the ignored-item suppression + move/hover ladder, over a
   TasksDragMoveSnapshot of presence/identity/launcher flags computed
   as TRIVIAL reads in QML (identity comparisons stay where object
   identity lives; `above === ignoredItem` covers JS null==null since
   both operands are QML nulls). Output enum SuppressRepeatTarget |
   MoveDragSource(moveTo) | KeepOrder | HoverAbove | ClearHover | None.
   target.animating pre-check and the preview-hide tail stay QML (live
   scenegraph/window reads).

Wrapper (new): declarativeimports/core/dropclassifiertools.{h,cpp},

## Faithful oddities preserved (pinned, not "fixed")

- urls.every() on an empty list is TRUE: a no-url, non-plasmoid drag
  classifies onlyLaunchers=true in BOTH consumers (Qt5 identical).
  Containment enter still rejects task drags before it matters; the
  plasmoid accepts such drags and urlsDropped([]) is a no-op.
- containment isPlasmoid requires the service-name format to be the
  FIRST formats entry (indexOf === 0), while isSeparator/isLatteTasks
  (mimeContainsPlasmoid) only require presence + a matching line.
- plasmoid separator check compares the WHOLE payload; a multi-line
  payload containing a separator line matches in the containment but
  not in the plasmoid.
- insertIndexAt returns stripe-1 which is -1 at distance 0 and more
  negative before the target origin; tasksModel.move rejects those.
- external-drag suppression: dragSource==null && ignoredItem==above
  compares null==null true, so an external drag over empty dock area
  early-returns before the ClearHover branch (hover clears on
  dragLeave instead).

## Eliminated invalid states (for the ledger tick at merge)

- A drag with no classification is unrepresentable at the disposition
  boundaries: every ladder takes the flags struct produced by the one
  classifier, so the Qt5 shape "flags assigned one by one with a timer
  racing them back to false" cannot hand a half-written state to a
  decision (QML still stores flags for its bindings, but decisions
  consume the classifier's return directly).
- The two consumers' accept sets can no longer drift apart silently:
  one header owns both rule sets side by side, and the truth-table
  suite runs the same mime vectors through both (the drift regression
  test the spec asked for).
- MoveDragSource without a target index is unrepresentable:
  TasksDragMoveDecision carries std::optional<int> moveTo and the
  factory asserts action==MoveDragSource <=> moveTo.has_value()
  (QT_FORCE_ASSERTS-live in tests).
- decideTasksDragMove's snapshot cannot express "drag source data on
  an absent drag source": dragSource is std::optional<DragSourceState>
  (index + isLauncher travel inside the optional), likewise above's
  model-attached flags live behind an optional.
- Wrapper boundary: null QMimeData, a non-callable predicate and
  itemStep <= 0 are refused loudly (qCritical + inert answer), never
  guessed around; a predicate JS exception is surfaced as a qWarning
  + url-not-application, not swallowed silently.
